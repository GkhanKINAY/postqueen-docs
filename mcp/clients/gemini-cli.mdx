title: Gemini CLI
description: 'Connect Gemini CLI to the PostQueen MCP server'
---

Gemini CLI supports MCP servers over streamable HTTP through its `settings.json` configuration, giving Gemini access to all 10 PostQueen tools from your terminal.

## Add via CLI

Grab your API key from [app.postqueen.ai](https://app.postqueen.ai) (**Settings**, then **Developers**, then **Public API**, click **Reveal**), then run:

```bash
gemini mcp add --transport http postqueen https://api.postqueen.ai/mcp/YOUR_API_KEY
```

Replace `YOUR_API_KEY` with your actual key. The command writes the server into your user config at `~/.gemini/settings.json` (pass `--scope project` to write to the project's `.gemini/settings.json` instead).

## Manual Configuration

You can also edit `~/.gemini/settings.json` directly. Gemini CLI uses the `httpUrl` key for streamable HTTP servers:

```json ~/.gemini/settings.json
{
  "mcpServers": {
    "postqueen": {
      "httpUrl": "https://api.postqueen.ai/mcp/YOUR_API_KEY"
    }
  }
}
```

<Note>
Keys per the official [Gemini CLI MCP documentation](https://github.com/google-gemini/gemini-cli/blob/main/docs/tools/mcp-server.md): `httpUrl` is the streamable HTTP endpoint (`url` is reserved for SSE servers — PostQueen does not use SSE). A project-level `.gemini/settings.json` uses the same format.
</Note>

## Bearer Variant

If you prefer to keep the key out of the URL, point `httpUrl` at the bare `/mcp` endpoint and pass the key in a `headers` object:

```json ~/.gemini/settings.json
{
  "mcpServers": {
    "postqueen": {
      "httpUrl": "https://api.postqueen.ai/mcp",
      "headers": {
        "Authorization": "Bearer YOUR_API_KEY"
      }
    }
  }
}
```

The equivalent one-liner:

```bash
gemini mcp add --transport http postqueen https://api.postqueen.ai/mcp --header "Authorization: Bearer YOUR_API_KEY"
```

<Warning>
Your API key grants full access to your PostQueen workspace — treat it like a password. Never commit a project-level `.gemini/settings.json` that contains the key, and keep the user-level file private.
</Warning>

## Self-Hosted

On a self-hosted instance, replace `https://api.postqueen.ai` with your `NEXT_PUBLIC_BACKEND_URL` in either form. Your reverse proxy must forward the `/mcp` paths and support streaming HTTP — see [Reverse Proxies](/reverse-proxies/caddy).

## Verify

Start Gemini CLI and run the built-in slash command:

```
/mcp
```

It lists configured servers with their connection status and discovered tools — `postqueen` should show as connected with 10 tools. Then ask:

> "List my connected social media accounts"

If the connection works, Gemini calls the `integrationList` tool and returns your connected accounts.
