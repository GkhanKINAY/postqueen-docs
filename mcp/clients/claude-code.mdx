title: Claude Code
description: 'Connect Claude Code to the PostQueen MCP server'
---

Claude Code connects to PostQueen over streamable HTTP with a single `claude mcp add` command, giving it access to all 10 PostQueen tools from your terminal.

## Add via CLI

Grab your API key from [app.postqueen.ai](https://app.postqueen.ai) (**Settings**, then **Developers**, then **Public API**, click **Reveal**), then run:

```bash
claude mcp add --transport http postqueen https://api.postqueen.ai/mcp/YOUR_API_KEY
```

Replace `YOUR_API_KEY` with your actual key. The server is registered for your local Claude Code configuration and is available in every session from now on.

## Project Config Alternative

To share the server with everyone working in a repository, declare it in a `.mcp.json` file at the project root instead (the same file `claude mcp add --scope project` writes):

```json .mcp.json
{
  "mcpServers": {
    "postqueen": {
      "type": "http",
      "url": "https://api.postqueen.ai/mcp/YOUR_API_KEY"
    }
  }
}
```

<Note>
Format per the official [Claude Code MCP documentation](https://code.claude.com/docs/en/mcp). Claude Code expands `${VAR}` environment variables in `.mcp.json`, so you can keep the key out of the file with `"url": "https://api.postqueen.ai/mcp/${POSTQUEEN_API_KEY}"`.
</Note>

## Bearer Variant

If you prefer to keep the key out of the URL, use the bare `/mcp` endpoint with an Authorization header:

```bash
claude mcp add --transport http postqueen https://api.postqueen.ai/mcp --header "Authorization: Bearer YOUR_API_KEY"
```

<Warning>
Your API key grants full access to your PostQueen workspace — treat it like a password, and never commit it to a shared `.mcp.json`.
</Warning>

## Self-Hosted

On a self-hosted instance, replace `https://api.postqueen.ai` with your `NEXT_PUBLIC_BACKEND_URL` in either form. Your reverse proxy must forward the `/mcp` paths and support streaming HTTP — see [Reverse Proxies](/reverse-proxies/caddy).

## Verify

Ask Claude Code:

> "List my connected social media accounts"

If the connection works, it calls the `integrationList` tool and returns your connected accounts. You can also run `claude mcp list` to confirm the server shows as connected.
