PostQueen speaks MCP over streamable HTTP, so any MCP-compatible client can connect to her with a single URL. Pick your client below for a step-by-step guide, or use the endpoints directly if you already know your way around MCP configs.

## Endpoints

The server accepts two connection forms. Both authenticate with the same API key.

<Tabs>
  <Tab title="API Key in URL">
    ```
    https://api.postqueen.ai/mcp/YOUR_API_KEY
    ```
  </Tab>
  <Tab title="Bearer Token">
    ```
    URL: https://api.postqueen.ai/mcp
    Authorization: Bearer YOUR_API_KEY
    ```
  </Tab>
</Tabs>

Your API key lives in the PostQueen app at [app.postqueen.ai](https://app.postqueen.ai): open **Settings**, then **Developers**, then **Public API**, and click **Reveal**.

## Self-Hosted

On a self-hosted instance, replace `https://api.postqueen.ai` with your `NEXT_PUBLIC_BACKEND_URL`:

```
https://your-postqueen-server.com/mcp/YOUR_API_KEY
```

Your reverse proxy must forward the `/mcp` paths to the backend and support streaming HTTP. See [Reverse Proxies](/reverse-proxies/caddy).

## Verify Connection

Once connected, ask your AI agent:

> "List my connected social media accounts"

If the connection works, the agent calls the `integrationList` tool and returns your connected accounts. If it fails, check that the API key was pasted in full and that your client uses the streamable HTTP transport.
