This page covers how the PostQueen codebase is organized and which commands you need for local development. If you want to work on her internals, start here.

## Set up your development environment

Follow [How to set up your development environment](/installation/development) to install prerequisites and get the stack running locally.

Requirements from the repository root `package.json`:

* Node.js 22.x (`>=22.12.0 <23.0.0`)
* pnpm 10.6.1 (declared via the `packageManager` field)

## Architecture overview

Before writing code, read the [architecture overview](/howitworks). It explains how the services fit together and how a post travels from the editor to a connected network.

## Repository overview

PostQueen is open source. The application source code lives at [github.com/GkhanKINAY/postqueen-app](https://github.com/GkhanKINAY/postqueen-app).

The repository is a monorepo managed with [pnpm workspaces](https://pnpm.io/workspaces). The workspace globs in `pnpm-workspace.yaml` are `apps/*` and `libraries/*`. A single `.env` file at the repository root is shared by all apps, which keeps local development and deployment configuration in one place. Copy `.env.example` to `.env` to start.

### Apps

| App | What it is |
| --- | --- |
| `apps/frontend` | Web app built with [Next.js](https://nextjs.org/) and [TailwindCSS](https://tailwindcss.com/) |
| `apps/backend` | API server built with [NestJS](https://nestjs.com/), organized into controllers, services, and repositories with DTOs |
| `apps/orchestrator` | [Temporal](https://temporal.io/) workflows and activities; this is the service that schedules and publishes posts |
| `apps/commands` | CLI tasks for administrative and maintenance operations |
| `apps/extension` | Chrome extension |
| `apps/sdk` | Source of the [`@postqueen/node`](https://www.npmjs.com/package/@postqueen/node) SDK |

Shared code lives under `libraries/`, including the NestJS libraries used by the backend and orchestrator and the shared React components used by the frontend.

### Database

The backend uses [Prisma](https://www.prisma.io/) as the ORM with [PostgreSQL](https://www.postgresql.org/) as the database. The schema lives at `libraries/nestjs-libraries/src/database/prisma/schema.prisma`.

### Scheduling and background work

Post scheduling is handled by Temporal. The orchestrator app registers the workflows and activities that fire at the scheduled time and publish each post. Redis is used for caching and queues, not for scheduling posts.

## Common commands

All commands run from the repository root with pnpm. The main scripts in `package.json` are:

* `pnpm run dev` - starts the frontend, backend, and orchestrator in parallel, together with the extension
* `pnpm run prisma-generate` - generates the Prisma client (also runs automatically after `pnpm install`)
* `pnpm run prisma-db-push` - pushes the Prisma schema to the database
* `pnpm run build` - builds the frontend, backend, and orchestrator

Useful narrower scripts:

* `pnpm run dev:frontend`, `pnpm run dev:backend`, `pnpm run dev:orchestrator` - run a single app in dev mode
* `pnpm run dev:docker` - starts local infrastructure from `docker-compose.dev.yaml`
* `pnpm run test` - runs the Jest test suite
