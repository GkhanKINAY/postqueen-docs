This page explains how PostQueen authenticates with each social network and how it stores and uses tokens and credentials. Use it to evaluate PostQueen for security, privacy, and platform reviews.

## How PostQueen connects

- **Open source and self-hostable.** PostQueen is an open-source social media scheduler that supports X, LinkedIn, Instagram, Bluesky, Mastodon, Discord, and 30+ networks. You can run it on your own infrastructure and inspect every line of code.
- **Official OAuth flows where the platform offers them.** Most networks connect through the platform's own OAuth screen: you log in on the platform's site, approve the requested scopes, and the platform issues the token. PostQueen receives only the scopes you approve.
- **API-based publishing and analytics.** Posts and analytics go through each platform's publishing and analytics APIs. PostQueen has no headless-browser automation and does not crawl or scrape content from social platforms.
- **Credentials only where OAuth is not available.** A few networks offer no OAuth for third-party schedulers. For those, PostQueen asks for the credential that the platform provides for API access, such as a Bluesky app password, a WordPress application password, or a Nostr key. It is stored encrypted and used only for that channel.
- **One cookie-based exception.** Skool has no public API, so it connects through the [PostQueen Chrome extension](/configuration/chrome-extension) using your own browser session. This method is opt-in, documented, and carries its own terms-of-service warning.
- **Encrypted token storage.** Like any scheduler, PostQueen stores the tokens and credentials for the channels you connect. They are encrypted at rest and used only to publish and fetch analytics on your behalf.

## What this means for you

- If you connect an OAuth-based account on the hosted service, the token is issued by the platform through its own OAuth screen, with only the scopes you approve.
- If you self-host, you control where tokens and post content live. See [Configuration reference](/configuration/reference) for storage and encryption settings, and [OAuth configuration](/configuration/oauth) for setting up your own provider credentials.
- If you build integrations on top of PostQueen, use the [Public API](/public-api/introduction) or [MCP server](/mcp/introduction). Both go through the same authenticated flows and respect the same platform limits.

## Reporting a concern

If you believe a PostQueen integration violates a platform's terms, or you have a security concern, email **support@postqueen.ai** with the details so the team can investigate.
